# 任务：PRD 多租户隔离与知识库权限补全

## 背景
当前 PRD（prd.md）在多租户隔离和知识库权限方面存在设计缺口。
平台方运营模式为：平台方是总控方，管理所有公共知识库、行业知识库和内置模板，
租户不可见也不可修改。企业知识库由租户自行上传和管理。系统在底层自动调用
平台知识库来补充企业知识库的不足。

## 待排查问题清单

### P0 - 必须解决
1. **租户对平台知识库的可见性约束**：PRD未显式声明租户不可见/不可改平台知识
2. **数据模型隔离设计**：平台级知识与企业级知识在pgvector中的存储隔离方案
3. **RLS策略具体规则**：行级安全策略的具体SQL实现
4. **知识入库目标锁定**：企业用户上传只能进入企业私有库，不能触及平台库
5. **前端权限边界**：Web知识管理界面只能操作企业私有库

### P1 - 需要补充
6. **素材检索Agent的三层检索权限控制**：确保检索时平台库数据对租户透明调用
7. **知识库管理Agent的入库权限校验**：Agent解析后只能写入企业私有库
8. **API层的知识库访问控制**：API接口层面的权限隔离
9. **平台方管理后台**：平台方维护公共/行业知识库的独立入口

## 预期交付物
- 更新后的 PRD 相关章节
- 数据模型设计（表结构 + 字段说明）
- RLS策略SQL
- 前端权限控制方案
- API权限隔离方案

## 约束
- 基于现有技术栈（PostgreSQL + pgvector + CrewAI + Streamlit/Next.js）
- 不改变已有的三层知识库架构，只补全隔离细节
- 保持与现有PRD其他章节的一致性
